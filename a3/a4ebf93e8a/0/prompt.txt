# Plan: Cleaning card — swap edge case, visual polish

## Context

Three improvements to the interactive cleaning schedule card:

1. **Compensation weeks are locked** — If a swap is created (e.g. Alice ↔ Bob), the compensation/return week shows "Upcoming" with no action buttons. If that person also can't do it, they're stuck with no escape hatch. Need a way to cancel the swap from the compensation side.

2. **Green left border when current week is done** — The current week already has a thick left border (`border-left: 4px solid var(--primary-color)`). When the shift is marked done, this should turn green (`--success-color`).

3. **Past weeks more transparent, undo button stays visible** — Past weeks use `opacity: 0.75` but the undo button inside should remain fully opaque/white so it's clearly actionable.

## File to modify

- `custom_components/hass_flatmate/frontend/hass-flatmate-cleaning-card.js`

## Changes

### 1. Allow canceling swaps from compensation weeks

The `canSwap` condition currently blocks compensation weeks:
```javascript
const canSwap = members.length > 1 && !isPast && !isDone && row.override_type !== "compensation";
```

**Approach**: Don't change `canSwap` (we don't want the full swap modal on compensation weeks). Instead, add a dedicated **"Cancel swap"** button on future compensation weeks. This is simpler than opening the swap modal — it directly cancels the linked swap pair.

**Button logic** (in the action buttons section, ~line 1172–1205):
- New condition: `const canCancelSwap = !isPast && !isDone && row.override_type === "compensation"`
- When `canCancelSwap` is true, render a "Cancel swap" button (neutral styling, `mdi:close-circle-outline` icon)
- On click → calls `_cancelLinkedSwap(row)` with `event.stopPropagation()`

**New method `_cancelLinkedSwap(row)`**:
- Find the source swap week using `row.source_week_start` — scan schedule weeks for the row with `override_type === "manual_swap"` and `week_start === row.source_week_start`
- Extract `originalAssigneeId` (= `baseline_assignee_member_id` of the source week) and `swappedWithId` (= `assignee_member_id` of the source week)
- Call `_swapWeek(sourceRow, originalAssigneeId, swappedWithId, true)` — reuses existing cancel logic
- This cancels the whole swap pair (both manual_swap + compensation revert)

### 2. Green left border for done current week

Add a CSS rule:
```css
.week-row.current.done {
  border-left-color: var(--success-color, #43a047);
}
```

### 3. Past week opacity with visible undo button

Replace the current `.week-row.past { opacity: 0.75; }` with a more targeted approach:

```css
.week-row.past .week-main {
  opacity: 0.55;
}
.week-row.past .week-actions .status-chip {
  opacity: 0.55;
}
```

This dims the text content and status chips but leaves action buttons (like undo) at full opacity. The undo button inherits the base `.action` styling which has `background: var(--card-background-color)` — stays crisp and clickable.
